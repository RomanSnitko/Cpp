## Контейнер, обеспечивающий доступ и вставку за O(1)

# Функциональные требования:
* Template-based: Кэш должен работать с любыми типами ключей и значений LRUCache<K, V>.
* Fixed Capacity: Размер задается при создании.
* 
Метод put(key, value):
* Если ключ есть — обновить значение и пометить как недавно использованное.
* Если ключа нет — добавить. Если кэш полон — удалить самый старый элемент.
*
Метод get(key):
* Возвращает значение.
* Помечает ключ как недавно использованный.
* Если ключа нет — бросает исключение или возвращает std::optional.
*
std::list — для хранения элементов в порядке использования.
*
std::unordered_map — для быстрого поиска места в списке по ключу.
